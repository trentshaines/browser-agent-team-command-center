# New Project Flow

End-to-end trace of what happens when a user clicks "New Project" and launches a task.

The flow is a two-step modal: the user describes their goal, Claude suggests an agent team, the user reviews/edits the team, then confirms to launch.

---

## Step 1: Modal Opens ‚Äî Enter Goal

**File:** `frontend/src/lib/components/CreateProjectModal.svelte`

The `+` button on `/james` sets `createModalOpen = true`, rendering `CreateProjectModal`.

**Step 1 shows a single field:**

| Field | Type | Binding | Placeholder |
|-------|------|---------|-------------|
| Goal | `<textarea>` (4 rows) | `prompt` | `"e.g. Compare prices for AirPods Pro across Amazon, Best Buy, and Walmart"` |

No project name field ‚Äî the title is auto-generated by the planning step.

The **Plan Team** button is disabled until the prompt is non-empty. Pressing `Cmd+Enter` / `Ctrl+Enter` also submits.

---

## Step 2: Plan the Agent Team

**File:** `frontend/src/lib/components/CreateProjectModal.svelte` ‚Äî `planTeam()`

When the user clicks **Plan Team**:

1. Sets `planning_loading = true` (shows spinner, button says "Planning...")
2. Calls `planning.create(prompt)` from `api.ts`

### API Call

**File:** `frontend/src/lib/api.ts`

```
POST /plan
Body: { "prompt": "<goal>" }
Response: { "title": "...", "agents": [{ "name": "...", "task": "..." }, ...] }
```

### Backend Planning Endpoint

**File:** `backend/app/routers/plan.py`

`POST /plan` uses the Anthropic API (Haiku for speed, or Bedrock if configured) to decompose the goal:

1. Sends the user's prompt to Claude with a system prompt that instructs it to return JSON with:
   - `title` ‚Äî a short 3-6 word project title
   - `agents` ‚Äî 1-5 agents, each with a `name` and specific `task`
2. Parses the JSON response
3. Returns a `PlanResponse` to the frontend

The system prompt includes examples so Claude understands the expected output format. It instructs agents to be parallelizable and specific.

---

## Step 3: Review & Edit Team

**File:** `frontend/src/lib/components/CreateProjectModal.svelte` ‚Äî step `'review'`

After planning succeeds, the modal transitions to the review step showing:

1. **Project Title** ‚Äî editable text input, pre-filled with Claude's suggestion
2. **Goal** ‚Äî read-only display of the original prompt
3. **Agent Team** ‚Äî list of agent cards, each with:
   - Numbered badge (1, 2, 3...)
   - Editable agent name
   - Editable task description textarea
   - Remove button (visible on hover, hidden if only 1 agent)
4. **"+ Add agent"** link to add a blank agent card
5. **Back** button to return to the prompt step
6. **Launch N Agents** button

The launch button is disabled unless every agent has both a name and a task filled in.

---

## Step 4: Launch ‚Äî Create Session + Spawn Agents

When the user clicks **Launch N Agents**:

**File:** `frontend/src/lib/components/CreateProjectModal.svelte` ‚Äî `launch()`

1. Creates a session: `sessionsApi.create(title)` ‚Üí `POST /sessions`
2. Calls `onLaunch(session.id, prompt, agents)` with the confirmed agent list

**File:** `frontend/src/routes/james/+page.svelte` ‚Äî `onProjectLaunched()`

3. Resets all UI state (messages, agent runs, frames)
4. Opens SSE connection: `connectSSE(id)` ‚Üí `GET /sessions/{id}/stream`
5. Calls direct task spawning: `tasks.spawn(id, goal, agents)` ‚Üí `POST /sessions/{id}/task`

### Backend: Create Session

**File:** `backend/app/routers/sessions.py`

```
POST /sessions
Body: { "title": "<auto-generated title>" }
```

Creates a `Session` row in the database with the planned title.

### Backend: Spawn Agents

**File:** `backend/app/routers/tasks.py`

```
POST /sessions/{id}/task
Body: {
  "prompt": "<goal>",
  "agents": [
    { "name": "Amazon Searcher", "task": "Go to amazon.com and..." },
    { "name": "Best Buy Searcher", "task": "Go to bestbuy.com and..." }
  ]
}
```

For each agent:
1. Creates an `AgentRun` record (status: `RUNNING`)
2. Registers in `agent_registry`
3. Publishes `agent_spawned` SSE event
4. Spawns `scripts/browser_agent.py` as a subprocess

All agents run in parallel via `asyncio.gather`.

---

## Step 5: SSE Streams Results to Frontend

**File:** `frontend/src/routes/james/+page.svelte`

The SSE connection (`EventSource`) listens for these event types:

| Event | What it does |
|-------|-------------|
| `delta` | Appends streamed text to the last assistant message |
| `done` | Marks streaming complete, finalizes agent frames |
| `agent_event` | On `agent_spawned`: adds a new agent tile with id, name, task |
| `agent_event` | On `agent_complete`: marks agent as done with result |
| `agent_log` | Appends a step log entry to the matching agent run |
| `agent_frame` | Updates the live screenshot/URL for an agent tile |
| `error_event` | Displays error state |
